Welcome to Q

Q is an online badminton queueing app. Given a set of players and a number of courts, a queue and corresponding matches can be generated.

It supports singles matches and doubles matches. Q keeps track of the players' statistics, and it also keeps a record of all matches played.

Please take the time to read the general instructions below. Most importantly, please read the notes on queues before creating a queue.

Players

In order to create a queue, the players to put in it must already exist. You can access the list of players on the right side of the navigation bar. To add a new player, simple click on the Player page's 'ADD PLAYER' button. It is just above the table of players. The player form asks for your name and your estimated level. The better the player, the higher your level should be. Upon creation, Q initializes the number of matches you've played in and the number of matches you've won to zero. A player's name can be changed, and a player's level can be updated. One can also view a player's details. A player can also be deleted. To do these, go to the Player page. The link is on the right side of the navigation bar. To view a player's detail, simply click their name. A player can be updated and deleted using the table of player's or using the player's detail page. These processes are illustrated below.

Queues

Creating a Queue

To create a queue, go to the Queue page via link on the right side of the navigation bar. This link goes to a listing of all queues made. You may click the date of the queue in order to view its details. To start a queue, you may either go to the current tab, or click the 'CREATE NEW QUEUE' button on the right side of the list tab.

When creating a queue, input the match types for all the matches the queue will generate, the number of courts, and the players involve. Please see the notes sections for some reminders when setting up the queue. Upon the creation of the queue, you will be redirected to the current tab of the Queue page. Should the queue you have created be successful, the current tab should display buttons for ending the queue and changing the courts, a table of players whose table header is a link to change court players, and a panel on the right side of the page for the matches generated. Notes on Queues

On Players

Players cannot be deleted while they are in queue. Players can be added to a queue at any point. Players can be removed from a queue as long a they are not in an ongoing match.

On Courts

You may change the number of courts in a queue at any given time. However, should all current courts in the queue have an ongoing match, you will be unable to change the number of courts to less than the current number. You must have at least the same number of courts with no ongoing matches as the number of courts you wish to remove.

Queues for Singles

When setting the number of courts in the queue, make sure you have enough players in the queue. In general, given N number of courts, make sure you have at least ((N * 2) + 2) players. Any less will result into an error.

Queues for Doubles

When setting the number of courts in the queue, make sure you have enough players in the queue. In general, given N number of courts, make sure you have at least ((N * 4) + 4) players. Any less will result into an error.
